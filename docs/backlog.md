# Backlog Проєкту TranslationApp

Цей документ містить план розробки системи перекладу тексту (Morse/Braille) згідно з навчальними ітераціями.

## Ітерація 1 (Lab 34) — Baseline
**Мета**: Створення фундаменту системи.
**Завдання**:
    Визначення базових інтерфейсів: `ITextConverter`, `ITranslationRepository`.
    Створення моделей даних: `TranslationRequest`, `TranslationResult`, `TranslationRecord`.
    Налаштування системи збірки та базової структури каталогів.

## Ітерація 2 (Lab 35) — Бізнес-логіка, Persistence, Запити
 **Мета**: Реалізація функціональності та збереження даних.
**Завдання**:
    Реалізація конкретних конвертерів (`MorseConverter`, `BrailleConverter`).
    Створення `TranslationAppService` для координації процесу перекладу.
    Впровадження `IConverterFactory` для дотримання принципу Open/Closed (OCP).
    Реалізація механізму збереження результатів у `InMemoryTranslationRepository`.
    Забезпечення можливості отримання історії перекладів.

## Ітерація 3 (Lab 36) — Quality Gate, Тести, Fault Handling
**Мета**: Забезпечення стабільності та якості коду.
**Завдання**:
    Покриття Unit-тестами основної логіки конвертації.
    естування сервісу за допомогою Mock-об'єктів для репозиторію.
    Впровадження обробки виключень (Fault Handling) для некоректних вхідних даних або відсутніх фабрик.
    Перевірка дотримання SOLID принципів (Quality Gate).

## Ітерація 4 (Lab 37) — Release, Документація, Демо
**Мета**: Фіналізація та презентація продукту.
**Завдання**:
    Підготовка технічної документації (README, діаграми класів).
    Оптимізація коду та видалення зайвих залежностей.
    Створення консольного інтерфейсу або тестів для демонстрації роботи (Demo).
    Фінальний реліз системи.
